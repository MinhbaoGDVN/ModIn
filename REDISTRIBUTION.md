# ModIn - Developer & Redistribution Guidelines

Thank you for your interest in ModIn.

ModIn is an open-source project. You are welcome to learn from it, modify it, redistribute it, and build upon it.

## You are allowed to

- Use ModIn for personal or commercial projects.
- Study the source code.
- Modify any part of the project.
- Reverse engineer the executable.
- Fork this repository.
- Redistribute original or modified versions.

## If you redistribute ModIn

Please follow these simple rules:

1. Clearly state that your version is modified.
2. List the files or components you changed.
3. Keep credits to the original ModIn project.
4. Do not claim your version is the official ModIn release.

Example:

Modified files:

- main.go
- switch.go
- ui.go

Changes:

- Added GUI
- Improved modpack switching
- Fixed duplicate detection

## What we encourage

We appreciate:

- Bug fixes
- Performance improvements
- New features
- Pull Requests
- Community forks

## What we do not support

Please do not use ModIn for:

- Malware
- Illegal activities
- Misleading users by pretending a modified build is an official release

Thank you for helping improve ModIn.
